## 1. Scope of Phase 3

### 1.1 داخل النطاق (In Scope)

| المجال | ما يجب تسليمه |
|--------|----------------|
| كيان الصندوق | إنشاء، تعديل، تفعيل، تعليق، إلغاء (قبل التشغيل)، إغلاق نهائي |
| الربط المحاسبي | ربط صندوق حي بحساب تفصيلي ASSET قابل للترحيل وفعّال؛ تفرّد الحساب |
| الأمناء | تعيين أمين أساسي / مراقب؛ تسليم عهدة وفق القواعد |
| الجلسات اليومية | فتح، CLOSING، إغلاق، إعادة فتح بضوابط |
| الجرد | جرد إغلاق + جرد مفاجئ؛ فئات عملة؛ اعتماد/رفض |
| تسوية الفروقات | مسودة قيد `ADJUSTMENT` مع `source_type=CASH_COUNT_VARIANCE` عند فرق ≠ 0 |
| الرصيد الدفتري | احتساب حي من قيود `POSTED` فقط (لا دفتر موازٍ) |
| إعدادات الفروقات | مفاتيح `cash_variance_gain_account_id` / `cash_variance_loss_account_id` |
| التقارير الأساسية | قائمة صناديق، جلسة يومية، فروقات جرد، ملخص أرصدة |
| الأمان | JWT + `ACCOUNTS` + Audit + تزامن متفائل |
| الدفعات الداخلية | 3.A → 3.B → 3.C → 3.D → 3.E وفق المواصفة v1.1 |
| التوافق المستقبلي | عقد منطقي للحركات النقدية (توثيق/خيارات) دون تنفيذ جدول حركات كامل |

## 2. Definition of Done

تُعدّ **المرحلة 3 منتهية بالكامل** عندما يتحقق كل ما يلي معاً:

1. **الدفعات 3.A–3.E** منفَّذة ومقبولة وفق معايير هذا المستند.  
2. جميع بنود جدول المتطلبات النهائي ذات الأولوية الملزمة (`Must`) بحالة **`Ready`**.  
3. لا انحدار على محرك القيود (المرحلة 2): إنشاء/اعتماد/ترحيل/عكس قيد يعمل كما قبل.  
4. الرصيد الدفتري للصندوق يطابق مجموع مدين−دائن لسطور `POSTED` على `account_id` المرتبط.  
5. لا يمكن إغلاق جلسة بلا جرد إغلاق معتمد (إلا مسار `bypass` النادر إن فُعّل ومُوثّق).  
6. فرق الجرد ≠ 0 يمر عبر قيد تسوية مرتبط بـ `CASH_COUNT_VARIANCE` (أو يُرفض الإغلاق).  
7. Audit يسجّل التغييرات الجوهرية (حالة الصندوق، جلسة، جرد، أمين، تسوية).  
8. واجهات `/accounts/cashbox*` تعمل RTL ومتسقة مع هوية نظام الحسابات.  
9. اختبارات القبول في الفصل 10 منفَّذة بنتيجة ناجحة للبنود الملزمة.  
10. تحديث تتبع المرحلة في الوثيقة المعمارية (رقم Migration + إشارة اكتمال المرحلة 3) بعد التنفيذ الفعلي.  
11. لا كود/جداول للسندات أو البنوك داخل نطاق «اكتمل المرحلة 3».

**Done الجزئي للدفعات:**

| دفعة | Done عند |
|------|----------|
| 3.A | CRUD صناديق + أمناء + إعدادات فروقات (مفاتيح) + قائمة/تفاصيل |
| 3.B | جلسة حية واحدة + جرد إغلاق مبسّط + إغلاق بفرق 0 فقط |
| 3.C | فئات + فرق ≠ 0 + قيد تسوية + رفض إغلاق بعد نشاط دفتري لاحق |
| 3.D | إعادة فتح + تعليق/إغلاق نهائي + تسليم عهدة + Audit كامل |
| 3.E | تقارير أساسية + options + توثيق عقد الحركات المستقبلية |

## 3. Functional Acceptance Criteria

### 3.1 إدارة الصندوق (كيان)

| ID | المعيار | Must/Should |
|----|---------|-------------|
| F-01 | إنشاء صندوق بحالات `DRAFT` مع كود، اسم عربي، نوع، حساب مؤهل | Must |
| F-02 | تعديل بيانات غير حرجة مع `version` | Must |
| F-03 | تفعيل إلى `ACTIVE` فقط بوجود أمين أساسي وحساب صالح | Must |
| F-04 | تعليق `SUSPENDED` ومنع جلسات جديدة | Must |
| F-05 | إعادة تفعيل من `SUSPENDED` → `ACTIVE` | Must |
| F-06 | إلغاء `CANCELLED` قبل تشغيل جوهري وتحرير الحساب | Must |
| F-07 | إغلاق نهائي `CLOSED` مع تحرير الحساب وحفظ `closed_account_id` | Must |
| F-08 | أنواع: `MAIN` / `FEES` / `PETTY` / `SUB` / `OTHER` | Must |
| F-09 | `PETTY` يرفض التفعيل/الحفظ بلا `ceiling_amount > 0` | Must |
| F-10 | عرض الرصيد الدفتري المحسوب (لا حقل إدخال رصيد حر) | Must |
| F-11 | منع تغيير `account_id` بعد أول جلسة تشغيلية | Must |

### 3.2 الأمناء

| ID | المعيار | Must/Should |
|----|---------|-------------|
| F-20 | تعيين أمين أساسي واحد ساري لكل صندوق | Must |
| F-21 | دعم دور `SUPERVISOR` | Should |
| F-22 | تسليم عهدة أثناء جلسة مفتوحة: جرد مفاجئ معتمد أو إغلاق أولاً | Must |
| F-23 | تسجيل تغيير الأمين في Audit | Must |

### 3.3 الجلسات

| ID | المعيار | Must/Should |
|----|---------|-------------|
| F-30 | فتح جلسة لصندوق `ACTIVE` فقط | Must |
| F-31 | إلزام `fiscal_year_id` + `fiscal_period_id` + `session_date` | Must |
| F-32 | السنة ACTIVE والفترة OPEN والتاريخ ضمن حدود الفترة | Must |
| F-33 | جلسة حية واحدة فقط (`OPEN`/`CLOSING`/`REOPENED`) لكل صندوق | Must |
| F-34 | لقطة `book_balance_at_open` عند الفتح | Must |
| F-35 | Submit جرد إغلاق ينقل الجلسة إلى `CLOSING` | Must |
| F-36 | رفض جرد إغلاق يعيد الجلسة إلى `OPEN` أو `REOPENED` | Must |
| F-37 | إغلاق الجلسة يحفظ لقطات الرصيد الفعلي/الدفتري/الفرق | Must |
| F-38 | إعادة فتح بسبب إلزامي + حالة `REOPENED` + زيادة `reopen_count` | Must |
| F-39 | منع فتح جلسة جديدة وجلسة سابقة حية (س10) | Must |

### 3.4 الجرد والتسوية

| ID | المعيار | Must/Should |
|----|---------|-------------|
| F-40 | إنشاء جرد `CLOSING` / `SURPRISE` | Must |
| F-41 | تعديل مسودة جرد؛ منع تعديل بعد الاعتماد | Must |
| F-42 | Submit يجمّد `book_balance` ويحسب `physical_total` و`variance` | Must |
| F-43 | فئات عملة قابلة للتهيئة + بنود بكميات ≥ 0 | Must |
| F-44 | اعتماد جرد بفرق 0 دون قيد تسوية | Must |
| F-45 | اعتماد جرد بفرق ≠ 0 ينشئ مسودة قيد `ADJUSTMENT` مرتبطة | Must |
| F-46 | القيد: `source_type=CASH_COUNT_VARIANCE` و`source_id=count.id` | Must |
| F-47 | اتجاه القيد وفق ربح/خسارة الفروقات من إعدادات السياسة | Must |
| F-48 | Idempotency: لا قيد ثانٍ لنفس الجرد | Must |
| F-49 | رفض إنشاء قيد تسوية إن حسابات الفروقات غير مهيأة | Must |
| F-50 | جرد مفاجئ لا ينقل إلى `CLOSING` ولا يغلق الجلسة | Must |
| F-51 | رفض إغلاق إن وُجدت قيود `POSTED` بعد لقطة الجرد | Must |
| F-52 | تحذير عند رصيد دفتري سالب دون منع | Should |

### 3.5 التقارير والخيارات

| ID | المعيار | Must/Should |
|----|---------|-------------|
| F-60 | قائمة صناديق مع رصيد وجلسة اليوم | Must |
| F-61 | تقرير/عرض جلسة يومية (فتح/إغلاق/فروقات) | Must |
| F-62 | قائمة فروقات الجرد مع مرجع القيد إن وُجد | Must |
| F-63 | ملخص أرصدة الصناديق | Must |
| F-64 | `options`: حسابات مؤهلة، أنواع، فئات عملة | Must |
| F-65 | توثيق عقد الحركة النقدية المستقبلية (تعليق/README أو قسم في options) | Should |

## 4. Business Rules Checklist

يُتحقق من كل قاعدة عند الاختبار؛ الفشل في أي `Must` يمنع Exit.

| BR | القاعدة | Must |
|----|---------|------|
| BR-01 | كود الصندوق فريد بدون حساسية حالة الأحرف | Must |
| BR-02 | الاسم العربي إلزامي | Must |
| BR-03 | النوع من القائمة المعتمدة فقط | Must |
| BR-04 | الحساب تفصيلي + ASSET + allow_posting + is_active | Must |
| BR-05 | لا مشاركة حساب بين صناديق حية (DRAFT/ACTIVE/SUSPENDED) | Must |
| BR-06 | CANCELLED يحرّر الحساب فوراً | Must |
| BR-07 | CLOSED يحرّر الحساب مع حفظ closed_account_id | Must |
| BR-08 | أمين أساسي إلزامي قبل ACTIVE | Must |
| BR-09 | سقف PETTY إلزامي و > 0 | Must |
| BR-10 | لا تغيير account_id بعد أول جلسة تشغيلية | Must |
| BR-11 | لا SUSPEND/إغلاق نهائي وجلسة حية | Must |
| BR-12 | لا تشغيل على صندوق غير ACTIVE | Must |
| BR-13 | جلسة حية واحدة كحد أقصى | Must |
| BR-14 | سنة/فترة/تاريخ جلسة إلزامية وصحيحة | Must |
| BR-15 | إغلاق يتطلب جرد إغلاق APPROVED (أو bypass موثّق) | Must |
| BR-16 | فتح/تغيير حالة الجلسة داخل Transaction مع قفل | Must |
| BR-17 | الرصيد الدفتري من POSTED فقط | Must |
| BR-18 | تجميد book_balance عند Submit | Must |
| BR-19 | رفض إغلاق عند نشاط دفتري بعد اللقطة | Must |
| BR-20 | فرق ≠ 0 يُسوَّى بقيد عبر حسابات السياسة قبل الإغلاق الافتراضي | Must |
| BR-21 | لا كميات فئات سالبة | Must |
| BR-22 | إعادة فتح: سبب + صلاحية أعلى + Audit | Must |
| BR-23 | لا حذف صندوق له جلسات؛ تعطيل/إغلاق فقط | Must |
| BR-24 | كل تغيير جوهري في financial_audit_log | Must |
| BR-25 | تسليم عهدة أثناء OPEN: جرد مفاجئ معتمد أو إغلاق أولاً | Must |
| BR-26 | لا كسر محرك القيود المرحلة 2 | Must |
| BR-27 | المستخدم من JWT فقط | Must |
| BR-28 | المبالغ NUMERIC(18,3) | Must |
| BR-29 | لا يُخزَّن current_balance كمصدر حقيقة | Must |
| BR-30 | لا إدخال رصيد افتتاحي حر من شاشة الصندوق | Must |

| أعمدة مبالغ NUMERIC(18,3) | Must |

### 5.3 سلامة البيانات بعد التنفيذ

| المعيار | مطلوب |
|---------|--------|

| RESTRICT على حذف حساب مربوط بصندوق حي | Must |

## 6. API Acceptance

القاعدة: `/api/accounts/cash-boxes…` مع `requireAccountsAccess`.

### 6.1 قائمة المسارات الملزمة

| الطريقة | المسار | نجاح متوقع | فشل متوقع (عينات) |
|---------|--------|------------|-------------------|
| GET | `/cash-boxes` | 200 قائمة + أرصدة | 401 |
| POST | `/cash-boxes` | 201/200 إنشاء | 400 تحقق؛ 409 كود/حساب |
| GET | `/cash-boxes/{id}` | 200 تفاصيل | 404؛ 401 |
| PUT | `/cash-boxes/{id}` | 200 مع version | 409 version؛ 409 تغيير حساب محظور |
| POST | `/cash-boxes/{id}/activate` | 200 ACTIVE | 409 بلا أمين/حساب |
| POST | `/cash-boxes/{id}/suspend` | 200 | 409 جلسة حية |
| POST | `/cash-boxes/{id}/close` | 200 CLOSED + تحرير حساب | 409 جلسة حية |
| GET/PUT | `/cash-boxes/{id}/custodians` | 200 | 409 تسليم غير صالح |
| GET | `/cash-boxes/{id}/sessions` | 200 | 401 |
| POST | `/cash-boxes/{id}/sessions/open` | 201/200 OPEN | 409 غير ACTIVE / جلسة حية / فترة |
| POST | `.../sessions/{sid}/close` | 200 CLOSED | 409 بلا جرد / نشاط بعد لقطة |
| POST | `.../sessions/{sid}/reopen` | 200 REOPENED | 400 بلا سبب؛ 409 خارج N يوم |
| POST | `.../sessions/{sid}/counts` | 201 مسودة جرد | 409 حالة جلسة |
| PUT | `/counts/{id}` | 200 مسودة | 409 غير DRAFT |
| POST | `/counts/{id}/submit` | 200 SUBMITTED + تجميد رصيد | 400 كميات سالبة |
| POST | `/counts/{id}/approve` | 200 + قيد عند فرق | 409 حسابات فروقات؛ 409 تكرار قيد |
| POST | `/counts/{id}/reject` | 200 REJECTED (إن وُجد المسار) | Should |
| GET | `/cash-boxes/reports/summary` | 200 | 401 |
| GET | `/cash-boxes/options` | 200 | 401 |
| GET/PUT | إعدادات فروقات الجرد (ضمن settings أو options) | 200 | 400 حساب غير مؤهل |

### 6.2 سياسة الأخطاء

| الرمز | متى |
|-------|-----|
| 400 | تحقق إدخال (سبب إعادة فتح ناقص، كميات سالبة…) |
| 401 | بلا JWT صالح |
| 403 | خارج نظام ACCOUNTS (إن طُبّق فصل أدق) |
| 404 | مورد غير موجود |
| 409 | تعارض عمل/حالة/تفرّد/version |
| 500 | خطأ غير متوقع — لا يكشف أسراراً |

الرسائل للمستخدم: عربية ومفهومة وفق سياسة النظام.

## 9. Performance Acceptance

سياق المرحلة 3: عدد صناديق قليل (عادة أقل من عشرات).
